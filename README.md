# ta-management-system
Teaching Assistant Recruitment and Selection Management System is a web app that helps teachers, students, and admins manage TA recruitment online. Teachers post openings and select applicants, students apply and track results, and admins manage data, making the process faster and more organized. 

API (app/api/subject/)
ไฟล์ภายในโฟลเดอร์นี้ทำหน้าที่เป็น Web API เพื่อให้บริการข้อมูลในรูปแบบ JSON (application/json) สำหรับนำไปแสดงผลบนหน้าเว็บ (Frontend) โดยมีการเรียกใช้งานเมธอดต่างๆ จากฝั่ง Controller
ประกอบด้วย 3 ไฟล์หลัก ดังนี้:

1. dbcourse.php (ดึงข้อมูลรายวิชาจากระบบจัดการ)
หน้าที่: ทำการเรียกข้อมูลรายวิชาทั้งหมด หรือรายวิชาที่กำหนดผ่าน AdminController
การทำงาน: * รับพารามิเตอร์ course_id ผ่านช่องทาง GET (ถ้ามี)
เรียกใช้ฟังก์ชัน getSubject() เพื่อดึงข้อมูลรายวิชา
ส่งคืนข้อมูลกลับไปในรูปแบบ JSON พร้อมสถานะ success

2. subject.php (ดึงรายการวิชาที่เปิดสอนตามปีการศึกษา)
หน้าที่: ดึงรายการรายวิชาที่เปิดสอนสำหรับปีการศึกษาและภาคเรียนในปัจจุบัน
การทำงาน:เรียกใช้งาน SubjectController
สร้างการเชื่อมต่อโดยดึง Access Token (getToken())
ตรวจสอบปีการศึกษาและภาคเรียนปัจจุบัน (getAcad())
ดึงรายการรายวิชาทั้งหมดของเทอมนั้นด้วยฟังก์ชัน getCourseList()
ส่งคืนรายการวิชาเป็น JSON

3. detail.php (ดึงข้อมูลรายละเอียดเชิงลึกของรายวิชา)
หน้าที่: ใช้สำหรับดึงข้อมูลรายละเอียดเฉพาะเจาะจงของรายวิชาใดรายวิชาหนึ่ง
การทำงาน:เรียกใช้งาน SubjectController
รับพารามิเตอร์รหัสวิชา course_id ผ่านช่องทาง GET
ทำการดึง Access Token และข้อมูลปีการศึกษา/ภาคเรียนปัจจุบัน เช่นเดียวกับไฟล์ subject.php
เรียกใช้ฟังก์ชัน getCourseDetail() โดยระบุรหัสวิชา เพื่อเจาะจงดึงเฉพาะข้อมูลของวิชานั้นๆ
ส่งคืนข้อมูลรายละเอียดเป็น JSON

โครงสร้างและระบบการทำงานฝั่ง Backend (Controllers & Config)
ระบบถูกพัฒนาด้วยรูปแบบ MVC-like โดยแยกส่วนประมวลผล (Controller) ออกจากส่วนแสดงผลและฐานข้อมูลอย่างชัดเจน เพื่อให้ง่ายต่อการดูแลรักษา โครงสร้างการทำงานหลักประกอบด้วย:
ส่วนตั้งค่าและเชื่อมต่อ (Configuration)
Database.php
ทำหน้าที่เชื่อมต่อกับฐานข้อมูล MySQL ผ่าน PDO (PHP Data Objects)
มีการดักจับข้อผิดพลาด (Exception) และจำกัดการสร้างคลาสเชื่อมต่อซ้ำ (Singleton-like behavior) เพื่อลดภาระการทำงานของเซิร์ฟเวอร์

ระบบจัดการสิทธิ์การเข้าใช้งาน (Authentication)
AuthController.php
Login & Logout: ตรวจสอบอีเมลและรหัสผ่าน (password_verify) พร้อมแยกเส้นทาง (Redirect) ไปยัง Dashboard ตามสิทธิ์ผู้ใช้ (admin, teacher, student)
Register: ระบบลงทะเบียนสำหรับนักศึกษา โดยมีเงื่อนไขบังคับให้ใช้อีเมลของมหาวิทยาลัย (@silpakorn.edu) เท่านั้น และต้องตั้งรหัสผ่าน 8 ตัวอักษรขึ้นไป
Change Password: ระบบเปลี่ยนรหัสผ่านสำหรับผู้ใช้ที่เข้าสู่ระบบแล้ว

ระบบจัดการสำหรับผู้ดูแลระบบ (AdminController)
AdminController.php
ทำหน้าที่จัดการข้อมูลพื้นฐาน (CRUD) ของระบบทั้งหมด
User Management: เพิ่ม ลบ แก้ไข ข้อมูลผู้ใช้งาน โดยมีการป้องกันการลบบัญชีตัวเอง และลบข้อมูลที่เกี่ยวโยงกันแบบ Cascade
Subject & Recruitment Management: จัดการข้อมูลรายวิชาและประกาศรับสมัครต่างๆ ที่อยู่ในระบบ

ระบบสำหรับอาจารย์ (TeacherController)
TeacherController.php
Recruitment Management: สร้าง, แก้ไข, ลบ ประกาศรับสมัครผู้ช่วยสอน (TA) รวมถึงการจัดการกลุ่มเรียน (Section) และโควตา
Applicant Management: ดูรายชื่อผู้สมัคร, ตรวจสอบไฟล์แนบ (เช่น เกรด), และอัปเดตสถานะ (อนุมัติ/ปฏิเสธ) พร้อมส่งแจ้งเตือน (Notification) ไปยังนักศึกษา
PDF Export: ฟังก์ชันส่งออกรายงานรายชื่อ TA ที่ผ่านการอนุมัติในรูปแบบ PDF (ใช้ไลบรารี mPDF และเทมเพลต FPDI) พร้อมสรุปชั่วโมงการทำงานและคำนวณค่าตอบแทน

ระบบสำหรับนักศึกษา (StudentController)
StudentController.php
Dashboard & Recruitment List: แสดงรายการวิชาที่เปิดรับสมัคร โดยมีระบบคำนวณชั้นปีของนักศึกษาอัตโนมัติ เพื่อคัดกรองวิชาที่นักศึกษาสามารถสมัครได้ (ต้องอยู่ปีเดียวกับวิชานั้นหรือสูงกว่า)
Application Process: ระบบยื่นใบสมัครเป็น TA สามารถเลือก Section ที่ต้องการ ยื่นไฟล์เกรด (PDF/Images) และตรวจสอบโควตาแบบ Real-time ป้องกันการสมัครเกิน
Profile & History: อัปเดตข้อมูลส่วนตัว อัปโหลดเรซูเม่ และดูประวัติ/สถานะการสมัครของตนเอง

ระบบซิงค์ข้อมูลรายวิชา (SubjectController)
SubjectController.php
ทำหน้าที่เชื่อมต่อกับ API ของมหาวิทยาลัย (reg6.su.ac.th) ผ่าน cURL
มีการประมวลผลการถอดรหัสข้อมูลรูปแบบพิเศษ (Base64 + Gzip decode) จาก API
Sync Data: ดึงข้อมูลปีการศึกษา ภาคเรียน รายวิชา และรายชื่ออาจารย์ผู้สอน มาอัปเดตหรือเพิ่มลงในฐานข้อมูลท้องถิ่น (Local Database) อัตโนมัติ เพื่อลดภาระการกรอกข้อมูลแบบ Manual

ระบบช่วยเหลือ (Helpers & Others)
NotificationHelper.php
ตัวช่วยจัดการระบบแจ้งเตือนภายในแอปพลิเคชัน (In-app Notification)
รองรับการบันทึกแจ้งเตือนลงฐานข้อมูล, ดึงรายการแจ้งเตือนที่ยังไม่ได้อ่าน, นับจำนวน, และอัปเดตสถานะว่าอ่านแล้ว

ReportController.php
โค้ดตัวอย่าง (Template/Helper) สำหรับการสร้างและส่งออกไฟล์ PDF เบื้องต้นโดยใช้ mPDF

ไฟล์ public/index.php (Main Router)
ไฟล์นี้คือหัวใจหลักของระบบที่ผู้ใช้ทุกคนจะเข้าถึงเป็นอันดับแรก ทำหน้าที่เสมือนพนักงานต้อนรับที่คอยตรวจสอบว่าผู้ใช้ต้องการไปที่หน้าไหน (ผ่านพารามิเตอร์ ?page=...) และเรียกใช้งานลอจิกที่ถูกต้อง
การตั้งค่าเริ่มต้น (Initialization):
เปิดใช้งาน session_start() เพื่อรองรับระบบ Login
เปิดการแสดงผล Error (error_reporting) เพื่อให้ง่ายต่อการ Debug ระหว่างพัฒนา
ระบบ Routing (switch...case): ตรวจสอบค่าจากตัวแปร $_GET['page'] เพื่อแบ่งโซนการทำงานออกเป็น 5 ส่วนหลัก:
Authentication Zone: จัดการหน้า Login, Register, Logout และกระบวนการยืนยันตัวตน
Admin Zone: ควบคุมเส้นทางสำหรับผู้ดูแลระบบ เช่น การจัดการผู้ใช้ (manage_users), รายวิชา (manage_subjects), และประกาศทั้งหมด
Teacher Zone: ควบคุมเส้นทางสำหรับอาจารย์ เช่น หน้า Dashboard, การสร้าง/แก้ไขประกาศ (teacher_create_recruitment), การดูรายชื่อผู้สมัคร (view_applicants), และการจัดการ TA
Student Zone: ควบคุมเส้นทางสำหรับนักศึกษา เช่น หน้าแสดงรายการประกาศ (student_recruitment_list), การกดยื่นใบสมัคร (student_apply_process), และจัดการโปรไฟล์ส่วนตัว
API & Utilities: เส้นทางสำหรับระบบหลังบ้าน เช่น การซิงค์ข้อมูลรายวิชาจากมหาวิทยาลัย (sync_subjects), การออกรายงาน PDF, และการคืนค่าเป็น JSON สำหรับ API (api_...)
Error 404 Handling: หากผู้ใช้เข้าสู่ page ที่ไม่มีอยู่ในระบบ จะมีการดักจับและแสดงหน้า 404 Not Found แจ้งเตือนผู้ใช้อย่างสวยงาม

2. ไฟล์ public/read_notification.php (Notification Handler)
ไฟล์สคริปต์ขนาดเล็กที่ทำหน้าที่จัดการ "สถานะการอ่าน" ของระบบแจ้งเตือน (Notifications) โดยเฉพาะ
การทำงาน:เมื่อผู้ใช้คลิกที่กล่องข้อความแจ้งเตือน ระบบจะยิง Request มาที่ไฟล์นี้พร้อมส่งค่า id ของการแจ้งเตือน และ link ปลายทางมาด้วย
สคริปต์จะทำการเชื่อมต่อฐานข้อมูล และอัปเดตสถานะของแจ้งเตือนนั้นให้เป็น "อ่านแล้ว" (is_read = 1)
หลังจากอัปเดตฐานข้อมูลเสร็จสิ้น ระบบจะทำการ Redirect (header("Location: ...")) นำทางผู้ใช้ไปยังหน้าเป้าหมายที่เชื่อมโยงกับแจ้งเตือนนั้นทันทีแบบไร้รอยต่อ (Seamless experience)

ส่วนแสดงผลของผู้ดูแลระบบ (Admin Views)
ไฟล์ในโฟลเดอร์ views/admin/ ทำหน้าที่เป็นหน้าจอ (User Interface) สำหรับผู้ใช้งานระดับ Admin เพื่อใช้ในการบริหารจัดการข้อมูลภาพรวมทั้งหมดของระบบ โดยแต่ละไฟล์ทำงานสอดคล้องกับ AdminController ดังนี้:
1. admin_dashboard.php (หน้าแรกของผู้ดูแลระบบ)
หน้าที่: เป็นหน้า Landing Page หรือหน้าหลักหลังจากที่ Admin ทำการเข้าสู่ระบบสำเร็จ
การทำงาน: แสดงภาพรวม (Overview) และสถิติต่างๆ ของระบบ เช่น จำนวนผู้ใช้งานทั้งหมด, จำนวนรายวิชา, หรือจำนวนประกาศรับสมัครที่เปิดอยู่ เพื่อให้ Admin สามารถดูความเคลื่อนไหวของระบบได้อย่างรวดเร็ว

2. manage_users.php (หน้าจัดการผู้ใช้งาน)
หน้าที่: หน้าจอสำหรับบริหารจัดการบัญชีผู้ใช้งานในระบบ (Admin, Teacher, Student)
การทำงาน:แสดงตารางรายชื่อผู้ใช้งานทั้งหมด พร้อมระบบค้นหา
มีฟอร์ม (Modal) สำหรับ เพิ่ม บัญชีผู้ใช้งานใหม่
มีปุ่มสำหรับ แก้ไข ข้อมูล (เช่น การเปลี่ยน Role หรืออัปเดตอีเมล/รหัสผ่าน)
มีปุ่มสำหรับ ลบ บัญชีผู้ใช้งานออกจากระบบ

3. manage_subjects.php (หน้าจัดการรายวิชา)
หน้าที่: หน้าจอสำหรับบริหารจัดการข้อมูลรายวิชาที่จะเปิดรับสมัคร TA
การทำงาน:แสดงรายการวิชาทั้งหมดที่ถูกบันทึกไว้ในฐานข้อมูล
มีปุ่ม "ซิงค์ข้อมูล" (Sync) เพื่อสั่งให้ระบบไปดึงข้อมูลรายวิชาและอาจารย์ผู้สอนล่าสุดจาก API ของมหาวิทยาลัยแบบอัตโนมัติ
รองรับการเพิ่ม แก้ไข หรือลบข้อมูลรายวิชาแบบ Manual ได้เช่นกัน

4. manage_recruitments.php (หน้าจัดการประกาศรับสมัคร)
หน้าที่: หน้าจอส่วนกลางสำหรับจัดการประกาศรับสมัครผู้ช่วยสอนทั้งหมด
การทำงาน:
ให้สิทธิ์ Admin ในการมองเห็นและจัดการประกาศรับสมัครทั้งหมดในระบบ (แม้จะเป็นประกาศที่สร้างโดยอาจารย์ท่านอื่น)
สามารถจัดการรายละเอียดของประกาศ (Title, Description), โควตาที่รับ, คุณสมบัติ (เกรดขั้นต่ำ), และกลุ่มเรียน (Section) ต่างๆ ได้

5. reports.php (หน้าออกรายงาน)
หน้าที่: หน้าจอสำหรับสรุปข้อมูลและออกรายงานเพื่อนำไปใช้งานต่อ
การทำงาน: * เป็นหน้า UI ที่ให้ Admin เลือกเงื่อนไขหรือเรียกดูข้อมูลสรุป (เช่น รายชื่อ TA ที่ผ่านการอนุมัติแยกตามรายวิชา)
เชื่อมโยงกับฟังก์ชันการ Export เป็นไฟล์ PDF เพื่อพิมพ์เป็นเอกสารเบิกจ่ายหรือทำรายงานสรุปประจำเทอม

ส่วนแสดงผลระบบยืนยันตัวตน (Authentication Views)
ไฟล์ในโฟลเดอร์ views/auth/ ทำหน้าที่เป็นส่วนติดต่อผู้ใช้งาน (User Interface) สำหรับระบบจัดการบัญชีและการเข้าสู่ระบบ โดยทำงานร่วมกับ AuthController ประกอบด้วยไฟล์หลักๆ ดังนี้:
1. login.php (หน้าเข้าสู่ระบบ)
หน้าที่: หน้าจอแรกสุดสำหรับผู้ใช้งานทุกระดับ (Admin, Teacher, Student) เพื่อยืนยันตัวตนเข้าสู่ระบบ
การทำงาน: * แสดงฟอร์มรับข้อมูล อีเมล (Email) และ รหัสผ่าน (Password)
มีการแสดงข้อความแจ้งเตือน (Error Message) กรณีที่ผู้ใช้กรอกข้อมูลไม่ถูกต้อง หรือไม่มีอีเมลนี้ในระบบ
มีลิงก์สำหรับกดไปยังหน้าลงทะเบียน (สำหรับนักศึกษาใหม่)

2. register.php (หน้าลงทะเบียนบัญชีใหม่)
หน้าที่: หน้าจอสำหรับนักศึกษาที่ต้องการสร้างบัญชีเพื่อเข้าใช้งานระบบ (สำหรับ Role: Student)
การทำงาน:แสดงฟอร์มให้กรอกข้อมูลส่วนตัวเบื้องต้น เช่น ชื่อ-นามสกุล, อีเมล, รหัสผ่าน และ ยืนยันรหัสผ่าน
มีการแจ้งเตือนเงื่อนไขข้อบังคับต่างๆ (อ้างอิงจากระบบหลังบ้าน) เช่น ต้องใช้อีเมลของมหาวิทยาลัย (@silpakorn.edu) และรหัสผ่านต้องมีความยาวไม่น้อยกว่า 8 ตัวอักษร
หากสมัครสำเร็จจะมีการแจ้งเตือนและพากลับไปที่หน้า Login

3. change_password.php (หน้าเปลี่ยนรหัสผ่าน)
หน้าที่: หน้าจอสำหรับผู้ใช้งานที่ล็อกอินเข้าระบบมาแล้ว และต้องการตั้งค่าความปลอดภัยของบัญชีตนเองใหม่
การทำงาน:แสดงฟอร์มให้ผู้ใช้กรอก รหัสผ่านปัจจุบัน (Current Password) เพื่อยืนยันตัวตนอีกครั้ง
ให้กำหนด รหัสผ่านใหม่ (New Password) และกรอกยืนยันซ้ำให้ตรงกัน
รองรับการแสดงผลข้อความแจ้งเตือน (Alert/Feedback) ทั้งในกรณีที่เปลี่ยนรหัสผ่านสำเร็จ หรือกรณีที่รหัสผ่านเดิมไม่ถูกต้อง

ส่วนแสดงผล navbar.php (แถบเมนูนำทางและระบบแจ้งเตือน)
หน้าที่: เป็นส่วนประกอบ UI หลัก (Navigation Bar) ที่จะถูกเรียกใช้และแสดงผลอยู่ด้านบนหรือด้านข้างของทุกๆ หน้าเว็บ เพื่อให้ผู้ใช้งานสามารถนำทางไปยังฟีเจอร์ต่างๆ ของระบบได้อย่างสะดวกรวดเร็ว
การทำงานหลัก:Dynamic Menu (แสดงผลเมนูตามสิทธิ์ผู้ใช้งาน): ระบบจะตรวจสอบสิทธิ์จาก $_SESSION['role'] เพื่อเรนเดอร์เมนูที่สอดคล้องกับผู้ใช้นั้นๆ เช่น:
Admin: จะเห็นเมนูการจัดการระบบ (จัดการผู้ใช้, รายวิชา, ประกาศทั้งหมด)
Teacher: จะเห็นเมนูประกาศของฉัน และการจัดการผู้ช่วยสอน (TA)
Student: จะเห็นเมนูค้นหาประกาศรับสมัคร และประวัติการยื่นใบสมัครของตนเอง
Notification Bell (ระบบแจ้งเตือนอัจฉริยะ): ทำงานร่วมกับ NotificationHelper โดยใช้ฟังก์ชัน countUnread() เพื่อดึงตัวเลขแจ้งเตือนใหม่มาแสดงเป็น Badge สีแดงบนไอคอนกระดิ่ง และเมื่อคลิกจะเรียก getUnread() เพื่อแสดงรายการแจ้งเตือนแบบ Dropdown ให้ผู้ใช้รับทราบผลการทำงานแบบ Real-time
User Dropdown (เมนูจัดการบัญชีส่วนตัว): แสดงชื่อผู้ใช้งานปัจจุบัน พร้อมเมนูสำหรับเข้าสู่หน้า โปรไฟล์ (Profile), เปลี่ยนรหัสผ่าน (Change Password), และปุ่ม ออกจากระบบ (Logout) เพื่อเคลียร์ Session อย่างปลอดภัย

ส่วนแสดงผลของนักศึกษา (Student Views)
ไฟล์ในโฟลเดอร์ views/student/ ทำหน้าที่เป็นส่วนติดต่อผู้ใช้งาน (User Interface) สำหรับผู้ใช้งานระดับ นักศึกษา (Student) เพื่อใช้ในการค้นหา สมัคร และติดตามผลการคัดเลือกเป็นผู้ช่วยสอน (TA) โดยทำงานประสานกับ StudentController ซึ่งประกอบด้วยไฟล์หลักๆ ดังนี้:
1. student_dashboard.php (หน้าแดชบอร์ดนักศึกษา)
หน้าที่: หน้าโฮมเพจหรือหน้าแรกหลังจากนักศึกษาเข้าสู่ระบบ เพื่อดูภาพรวมกิจกรรมของตนเอง
การทำงาน:แสดงสถิติภาพรวม (Stats): สรุปข้อมูลตัวเลขที่สำคัญ ได้แก่ จำนวนวิชาที่เปิดรับและตรงกับชั้นปีของตนเอง (Eligible), จำนวนที่ยื่นสมัครไปแล้ว (Applied), และจำนวนที่ผ่านการคัดเลือก (Accepted)
ประกาศล่าสุด (Latest Announcements): แสดงรายการประกาศรับสมัคร TA ที่เพิ่งเปิดรับล่าสุด (แสดงผลสูงสุด 5 รายการ) เพื่อให้นักศึกษาไม่พลาดข่าวสารใหม่ๆ และสามารถกดสมัครจากหน้านี้ได้ทันที
ประวัติล่าสุด (Recent History): แสดงสถานะการสมัคร 5 รายการล่าสุด เพื่อให้ติดตามผลได้อย่างรวดเร็ว

2. recruitment_list.php (หน้ารายการประกาศรับสมัคร)
หน้าที่: หน้าจอหลักสำหรับการค้นหาและเลือกดูประกาศรับสมัคร TA ทั้งหมดที่เปิดทำการอยู่
การทำงาน:ระบบคัดกรองอัจฉริยะ: แสดงเฉพาะรายวิชาที่นักศึกษาสามารถสมัครได้ (ระบบจะคำนวณชั้นปีของนักศึกษา และแสดงเฉพาะวิชาของชั้นปีตัวเองหรือชั้นปีที่ต่ำกว่าเท่านั้น)
ระบบค้นหา (Search): สามารถพิมพ์ค้นหาชื่อวิชา รหัสวิชา หรือชื่อประกาศได้
ระบบยื่นใบสมัคร (Apply Modal): เมื่อกดปุ่ม "สมัคร TA" จะมีหน้าต่าง Pop-up ให้เลือกกลุ่มเรียน (Section) ที่ต้องการสมัคร (เลือกได้มากกว่า 1 กลุ่ม), กรอกเกรดรายวิชานั้นๆ, และอัปโหลดไฟล์หลักฐานผลการเรียน (PDF หรือรูปภาพ)
ตรวจสอบโควตา: แสดงจำนวนรับสมัคร (Quota) ของแต่ละ Section และมีระบบป้องกันการกดสมัครซ้ำ

3. my_applications.php (หน้าประวัติและสถานะการสมัคร)
หน้าที่: หน้าจอสำหรับติดตามสถานะใบสมัครทั้งหมดที่นักศึกษาเคยยื่นไป
การทำงาน:แสดงตารางประวัติการสมัคร พร้อมบอกรายละเอียดเช่น วิชา, อาจารย์ผู้สอน, Section ที่เลือก และวันที่ทำการสมัคร
ป้ายกำกับสถานะ (Status Badges): แสดงสถานะใบสมัครอย่างชัดเจนด้วยสีต่างๆ เช่น สีเหลือง (รอพิจารณา - Pending), สีเขียว (ผ่านการคัดเลือก - Approved), และสีแดง (ไม่ผ่านการคัดเลือก - Rejected)
ระบบยกเลิก (Cancel Application): มีปุ่มให้กดยกเลิกใบสมัครได้ด้วยตนเอง (อนุญาตให้ยกเลิกได้เฉพาะใบสมัครที่สถานะยังเป็น "รอพิจารณา" เท่านั้น)

4. profile.php (หน้าจัดการข้อมูลส่วนตัว)
หน้าที่: หน้าจอสำหรับดูและอัปเดตข้อมูลเรซูเม่ (Resume) และประวัติส่วนตัว
การทำงาน:ให้นักศึกษาสามารถแก้ไข ชื่อ-นามสกุล และ รหัสนักศึกษา ได้ (ส่วนอีเมลจะถูกล็อกไว้เนื่องจากใช้เป็นข้อมูลยืนยันตัวตนหลัก)
อัปโหลดเรซูเม่ (Resume Upload): รองรับการอัปโหลดไฟล์ประวัติส่วนตัวหรือแฟ้มสะสมผลงาน (PDF/Word/Images) เก็บไว้ในระบบ เพื่อให้อาจารย์สามารถคลิกดูประกอบการพิจารณารับเข้าทำงานได้ง่ายขึ้น

ส่วนแสดงผลของอาจารย์ (Teacher Views)
ไฟล์ในโฟลเดอร์ views/teacher/ ทำหน้าที่เป็นส่วนติดต่อผู้ใช้งาน (User Interface) สำหรับผู้ใช้งานระดับ อาจารย์ (Teacher) เพื่อใช้ในการเปิดรับสมัครผู้ช่วยสอน, คัดเลือกนักศึกษา, และจัดการรายชื่อ TA ที่ผ่านการอนุมัติแล้ว โดยทำงานร่วมกับ TeacherController ซึ่งประกอบด้วยไฟล์หลักๆ ดังนี้:
1. teacher_dashboard.php (หน้าแดชบอร์ดอาจารย์)
หน้าที่: หน้าจอแรกหลังจากอาจารย์เข้าสู่ระบบ เพื่อดูภาพรวมของการเปิดรับสมัครทั้งหมด
การทำงาน: * แสดงสถิติเบื้องต้น เช่น จำนวนประกาศรับสมัครที่เปิดอยู่ และจำนวนใบสมัครที่รอการพิจารณา (Pending)
แสดงรายการประกาศรับสมัครล่าสุด พร้อมปุ่มคีย์ลัดสำหรับเข้าไปจัดการข้อมูลหรือดูรายชื่อผู้สมัครได้อย่างรวดเร็ว

2. my_recruitments.php (หน้าจัดการประกาศรับสมัครของฉัน)
หน้าที่: หน้าจอหลักสำหรับจัดการประกาศรับสมัครผู้ช่วยสอน (TA) ของอาจารย์ท่านนั้นๆ
การทำงาน:สร้างประกาศใหม่ / แก้ไข: มีฟอร์มให้กรอกรายละเอียดวิชา, เกรดขั้นต่ำที่ต้องการ, และสามารถเพิ่ม/ลด กลุ่มเรียน (Section) พร้อมกำหนดจำนวนโควตาที่รับในแต่ละ Section ได้
ลบประกาศ: สามารถกดลบประกาศรับสมัครของตนเองได้ (ระบบจะจัดการลบข้อมูลที่เกี่ยวข้องในฐานข้อมูลให้ด้วย)
ระบบตัวกรอง: สามารถกรองดูประกาศตามภาคการศึกษา (Semester) ได้

3. view_applicants.php และ recruitment_applicants.php (หน้าจัดการผู้สมัคร)
หน้าที่: หน้าจอสำหรับตรวจสอบและคัดเลือกนักศึกษาที่ยื่นใบสมัครเข้ามา
การทำงาน:view_applicants.php: แสดงรายชื่อผู้สมัครทั้งหมด หรือภาพรวมของการสมัครในวิชาต่างๆ พร้อม Modal สำหรับดูรายละเอียดเชิงลึกของผู้สมัคร (เช่น เกรดที่ได้, กลุ่มเรียนที่เลือก) และมีปุ่มให้กดเปิดดูไฟล์เรซูเม่ (Resume/PDF) ที่นักศึกษาแนบมา
recruitment_applicants.php: ใช้สำหรับพิจารณาอนุมัติรายบุคคล โดยมีปุ่ม "อนุมัติ (Approve)" และ "ปฏิเสธ (Reject)" * ระบบป้องกันโควตาเต็ม: เมื่อกดอนุมัติ ระบบจะมีการตรวจสอบจำนวนรับ (Quota) หากรับ TA ครบตามจำนวนที่ตั้งไว้แล้ว ระบบจะแจ้งเตือนและไม่อนุญาตให้รับเพิ่ม

4. my_tas.php (หน้าจัดการผู้ช่วยสอนที่ผ่านการคัดเลือก)
หน้าที่: หน้าจอแสดงผลและจัดการรายชื่อนักศึกษาที่ ผ่านการอนุมัติให้เป็น TA แล้ว
การทำงาน:แสดงรายชื่อนักศึกษาโดยจัดกลุ่มตาม "รายวิชา" และ "กลุ่มเรียน (Section)" อย่างเป็นระเบียบ
มีปุ่มสำหรับ ถอดถอน (Remove/Revoke) สิทธิ์การเป็น TA ของนักศึกษาในกรณีที่มีการเปลี่ยนแปลงหรือข้อผิดพลาด โดยมี Pop-up ยืนยันความปลอดภัยก่อนดำเนินการ

5. ส่วนออกรายงานและพิมพ์เอกสาร (report_my_tas.php & template.php)
หน้าที่: เทมเพลตสำหรับการสร้างเอกสารเพื่อใช้ในการส่งเบิกจ่ายหรือทำรายงานสรุป
การทำงาน:report_my_tas.php: เทมเพลต HTML/CSS สำหรับสรุปรายชื่อ TA ที่อาจารย์รับเข้าทำงาน แยกตามรายวิชา เพื่อใช้ร่วมกับไลบรารีออกใบรายงาน (PDF)
template.php: เป็นฟอร์มหน้าตาเหมือนเอกสารราชการ/เอกสารมหาวิทยาลัย ("แบบฟอร์มแจ้งความประสงค์ในการจ้างนักศึกษาช่วยสอน") ที่เตรียมไว้สำหรับการดึงข้อมูลลงไปใส่ และพิมพ์ออกมาเป็นเอกสารส่งให้ทางภาควิชาหรือคณะต่อไป

สภาพแวดล้อมจำลองและการจัดการไลบรารี (Environment & Dependencies)
โปรเจกต์นี้ถูกออกแบบมาให้สามารถติดตั้งและรันบนเครื่องใดก็ได้ (Cross-platform) โดยไม่ต้องยุ่งยากกับการตั้งค่าเซิร์ฟเวอร์ด้วยตนเอง ผ่านการใช้งาน Docker และ Composer โดยมีไฟล์ตั้งค่าที่สำคัญดังนี้:

1. ไฟล์จัดการ Docker (Containerization)
Dockerfile
เป็นไฟล์คำสั่งสำหรับสร้าง Image ของ Web Server โดยใช้ PHP 8.2 (Apache) เป็นฐาน
การติดตั้ง Extensions: ทำการติดตั้งส่วนเสริมของ PHP ที่จำเป็นต่อโปรเจกต์ เช่น pdo_mysql (สำหรับเชื่อมต่อและจัดการฐานข้อมูล) และ gd (สำหรับการจัดการรูปภาพ)
Document Root: ตั้งค่าและชี้ให้ Apache โหลดหน้าเว็บเริ่มต้นจากโฟลเดอร์ /public เพื่อความปลอดภัย (ป้องกันไม่ให้ผู้ใช้เข้าถึงไฟล์ระบบหรือ Core logic ในโฟลเดอร์ app/ ได้โดยตรง)
docker-compose.yml เป็นไฟล์ที่ใช้ควบคุมและเชื่อมโยงคอนเทนเนอร์ (Containers) หลายๆ ตัวให้ทำงานร่วมกัน โดยในโปรเจกต์นี้ประกอบด้วย 3 เซอร์วิสหลัก:
app: คอนเทนเนอร์รันโค้ด PHP/Apache (ทำการ Map พอร์ตเพื่อให้เข้าถึงผ่าน http://localhost:8082 ได้)
db: คอนเทนเนอร์ฐานข้อมูล MySQL 8.0 (ตั้งค่ารหัสผ่าน root และสร้างฐานข้อมูลเริ่มต้นชื่อ TA1)
phpmyadmin: คอนเทนเนอร์สำหรับจัดการฐานข้อมูลผ่านหน้าต่างเบราว์เซอร์ (เข้าถึงผ่านพอร์ต 8081)

2. ไฟล์จัดการไลบรารีของ PHP (Package Management)
composer.json และ composer.lock
เป็นไฟล์ที่ใช้ระบุและล็อกเวอร์ชันของไลบรารี (Dependencies) ภายนอกที่โปรเจกต์นี้ต้องเรียกใช้งาน
แพ็กเกจที่สำคัญในระบบ:
mpdf/mpdf: ไลบรารีสำหรับสร้างและ Render ไฟล์เอกสาร PDF จากโค้ด HTML (ใช้ในส่วนที่อาจารย์ส่งออกรายงานรายชื่อ TA)
setasign/fpdi: ไลบรารีสำหรับอ่านและนำเข้าเทมเพลต PDF ที่มีอยู่แล้ว (เช่น ฟอร์มเอกสารของมหาวิทยาลัย) เพื่อนำมาเขียนข้อมูลทับซ้อนลงไป
